## **1. Connecting to MongoDB**

```python
from pymongo import MongoClient

# Connect to MongoDB (default: localhost:27017)
client = MongoClient("mongodb://localhost:27017/")

# Create or connect to a database
db = client["mydatabase"]

print("Connected to MongoDB!")
```

ğŸ”¹ **If the database doesn't exist, MongoDB creates it when you insert data.**

---

## **2. Creating a Collection (Equivalent to SQL Tables)**

```python
# Create or connect to a collection
collection = db["users"]

print("Collection created!")
```

ğŸ”¹ **Collections in MongoDB are like SQL tables but without a fixed schema.**

---

## **3. Inserting Documents (Equivalent to SQL INSERT)**

### **ğŸ‘‰ Insert a Single Document**

```python
user = {"name": "Alice", "email": "alice@example.com", "age": 28}
inserted_id = collection.insert_one(user).inserted_id

print(f"Inserted user with ID: {inserted_id}")
```

### **ğŸ‘‰ Insert Multiple Documents**

```python
users = [
    {"name": "Bob", "email": "bob@example.com", "age": 32},
    {"name": "Charlie", "email": "charlie@example.com", "age": 25}
]

inserted_ids = collection.insert_many(users).inserted_ids
print(f"Inserted multiple users with IDs: {inserted_ids}")
```

ğŸ”¹ **Each document automatically gets a unique `_id` field unless specified.**

---

## **4. Retrieving Data (Equivalent to SQL SELECT)**

### **ğŸ‘‰ Retrieve One Document**

```python
user = collection.find_one({"name": "Alice"})
print(user)  # Output: {'_id': ObjectId(...), 'name': 'Alice', 'email': 'alice@example.com', 'age': 28}
```

### **ğŸ‘‰ Retrieve All Documents**

```python
for user in collection.find():
    print(user)
```

### **ğŸ‘‰ Retrieve Specific Fields**

```python
for user in collection.find({}, {"_id": 0, "name": 1, "email": 1}):
    print(user)
```

ğŸ”¹ **Setting `_id: 0` excludes it from results.**

### **ğŸ‘‰ Using Query Filters**

```python
# Find users older than 30
for user in collection.find({"age": {"$gt": 30}}):
    print(user)
```

ğŸ”¹ **MongoDB uses `$gt`, `$lt`, `$gte`, `$lte` for comparisons.**

---

## **5. Updating Documents (Equivalent to SQL UPDATE)**

### **ğŸ‘‰ Update a Single Document**

```python
collection.update_one({"name": "Alice"}, {"$set": {"age": 29}})
```

### **ğŸ‘‰ Update Multiple Documents**

```python
collection.update_many({"age": {"$lt": 30}}, {"$set": {"status": "young"}})
```

ğŸ”¹ **Use `$set` to update specific fields.**

---

## **6. Deleting Documents (Equivalent to SQL DELETE)**

### **ğŸ‘‰ Delete a Single Document**

```python
collection.delete_one({"name": "Alice"})
```

### **ğŸ‘‰ Delete Multiple Documents**

```python
collection.delete_many({"age": {"$lt": 25}})
```

ğŸ”¹ **Use `delete_many()` with filters to remove multiple records.**

---

## **7. Using Indexes for Faster Queries**

```python
collection.create_index("email", unique=True)
```

ğŸ”¹ **Indexes improve search performance and enforce uniqueness (like SQL constraints).**

---

## **8. Handling MongoDB Connections Efficiently**

```python
try:
    client = MongoClient("mongodb://localhost:27017/", serverSelectionTimeoutMS=5000)
    client.server_info()  # Check connection
    print("Connected to MongoDB!")
except Exception as e:
    print(f"Error: {e}")
finally:
    client.close()
```

ğŸ”¹ **Always close the connection in a `finally` block.**

---

## **9. Using Connection Pooling**

```python
client = MongoClient("mongodb://localhost:27017/", maxPoolSize=50)
```

ğŸ”¹ **This optimizes performance in high-traffic applications.**

---

## **10. Aggregation (Advanced Queries)**

```python
pipeline = [
    {"$group": {"_id": "$age", "count": {"$sum": 1}}}
]

result = collection.aggregate(pipeline)
for doc in result:
    print(doc)
```

ğŸ”¹ **Aggregation is used for complex queries like grouping and summing.**
